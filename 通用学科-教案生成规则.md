# 教案生成 Skill（通用学科版）

## 概述

本 Skill 用于生成各学科通用的教学设计（HTML格式），基于标准教学设计表格结构，适用于中小学各学科、各年级。参考模板 `通用学科-教学设计参考模板.html`。

## 适用场景

当用户要求生成教学设计、编写教案、制作课堂教学设计时激活此 Skill。

---

## 一、文件位置

- **参考模板**：`.\通用学科-教学设计参考模板.html`
- **输出目录**：`.\`
- **命名规则**：`{课程名称}-{课题名称}-教学设计.html`

## 三、HTML表格结构（6列固定布局）

表格为6列表格，`table-layout: fixed`，宽度 `720px`。每行严格按照以下顺序：

### 基础信息行

```
行1: 授课时间 | 时间内容（colspan=2） | 课时  | 课时内容（colspan=2）
行2: 课题名称 | 课题内容（colspan=2） | 课型  | 课型内容（colspan=2）
行3: 授课班级 | 班级内容（colspan=2） | 授课教师 | 教师内容（colspan=2）
```

### 内容区块结构

每个内容区块格式：
```
标题行: section-title（colspan=6，居中加粗灰色背景）
内容行: label-cell（width=90px，加粗居中灰色背景） | 内容（colspan=5）
```

| # | 行内容 | 说明 |
|---|--------|------|

## 四、生成规则

### 1. 通用标题
- 页面 `<title>`：`{课程名称} - {课题名称} 教学设计`
- 页面主标题（`.title`）：`《{课程名称}》教学设计`

### 2. 授课时间行
- 格式：`____ 年 __ 月 __ 日    第 __ 周    星期 __    第 __ 节`
- 根据用户指定日期填写

### 3. 课题名称行
- 直接填写用户指定的课题名称
- 课型填写用户指定的课型

### 4. 授课班级 & 授课教师行
- 按用户提供的信息填写，可留空

### 5. 教学目标
- **维度可配置**：默认使用"知识与技能 / 过程与方法 / 情感态度与价值观"
- 也可替换为其他框架，如：
  - 核心素养四维度：语言建构与运用 / 思维发展与提升 / 审美鉴赏与创造 / 文化传承与理解
  - 三维目标新版：知识理解 / 能力发展 / 价值观念
  - 或用户自定义的其他维度名称和数量
- 每个维度一行，内容使用 `1. 2. 3.` 编号
- 核心公式/概念使用 `<strong>` 加粗

### 6. 教学重点
- 填写用户指定的重点
- 如果用户未明确，可根据知识点推断

### 7. 教学难点
- 填写用户指定的难点
- 如果用户未明确，可根据知识点推断

### 8. 教法（至少2种）
常用教法建议：
- 讲授法、演示法、实验法、讨论法
- 启发式教学、探究式教学、问题驱动法
- 任务驱动法、情境教学法、小组合作法
- 讲练结合法、多媒体辅助教学

### 9. 学法
常用学法建议：
- 自主探究法、合作学习法、观察比较法
- 练习巩固法、数形结合法、归纳总结法
- 体验学习法、迁移学习法

### 10. 可选模块A（如课程思政/德育渗透/核心素养等）
- **是否启用**：根据用户配置决定
- **模块名称**：使用用户指定的名称
- 内容根据学科特点设计，示例：
  - 语文：文化自信、家国情怀、审美教育
  - 数学：科学精神、理性思维、数学文化
  - 英语：跨文化意识、国际视野
  - 物理/化学：科学态度、社会责任、安全环保
  - 历史：唯物史观、家国情怀、历史担当

### 11. 可选模块B（如安全教育/实验规范/学科特色等）
- **是否启用**：根据用户配置决定
- **模块名称**：使用用户指定的名称
- 内容根据学科特点设计

### 12. 教学准备
- **教师**：多媒体课件、教具、实验器材、模型、学案等
- **学生**：预习要求、复习内容、需准备的工具等

### 13. 板书设计
包含：
- 课题标题
- 主要知识点（层次清晰）
- 核心公式/概念（使用 `<strong>` 加粗）
- 可使用 `white-space: pre-line` 保持格式

### 14. 教学实施（核心部分）

**环节数量**：5-7个环节

**每个环节包含**：
- 教学环节（含时间分配，如 `一、复习导入<br>（5分钟）`）
- 教学内容（colspan=2）
- 教师活动
- 学生活动
- 设计意图

**活动列宽度**：使用 CSS class `activity-cell`，宽度 150px

**通用推荐环节结构**：
1. **复习/导入**（3-5分钟）→ 复习旧知识/情境导入，引出新课
2. **新知探究**（12-18分钟）→ 核心知识点讲解、概念建构
3. **深化/应用**（10-15分钟）→ 公式推导/规律应用/例题讲解
4. **巩固练习**（8-10分钟）→ 典型例题 + 变式训练/小组讨论
5. **拓展延伸**（3-5分钟）→ 实际应用、学科联系、素养提升
6. **课堂小结**（2-3分钟）→ 知识回顾与梳理
7. **作业布置**（1-2分钟）→ 分层作业布置

（可根据学科和课型灵活调整环节名称和数量）

### 15. 课堂小结
- 列出本课主要知识点（带编号）
- 核心公式/概念加粗

### 16. 作业设计
建议分层设计：
- **基础巩固**：必做，全体学生完成
- **拓展提高**：选做，学有余力的学生完成
- **思考探究**：开放性问题，培养思维能力

### 17. 教学反思
- **预设问题及预案**：列出可能的问题及处理方案
- **亮点设计**：教学设计的创新和特色

## 五、格式规范

### 公式/符号
- 数学、物理等学科的公式使用标准格式
- 核心公式/概念使用 `<strong>` 标签包裹
- 乘号使用 `·` 或省略，不用 `*`
- 下标格式：`X₁`、`a_n` 等

### 换行
- 使用 `<br>` 换行
- 板书设计区域可使用 `white-space: pre-line`

---

## 六、HTML样式规范

### 页面布局
- 文档容器：`.doc-container`，宽度 `720px`，居中，内边距 `15px`
- 标题：`.title`，居中，`16pt`，加粗

### 表格样式
- 表格宽度 `720px`，边框合并，固定布局
- 单元格边框：`1px solid #000`
- 单元格内边距：`4px 6px`
- 字体：`12pt`，宋体
- 垂直对齐：顶部对齐

### CSS类说明
```html
<style>
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body { font-family: "宋体", SimSun, serif; font-size: 12pt; line-height: 1.5; }
    .doc-container { width: 720px; margin: 0 auto; padding: 15px; }
    .title { text-align: center; font-size: 16pt; font-weight: bold; margin-bottom: 8px; }
    table { width: 720px; border-collapse: collapse; table-layout: fixed; }
    td, th { border: 1px solid #000; padding: 4px 6px; font-size: 12pt; vertical-align: top; word-wrap: break-word; }
    .label-cell { width: 90px; font-weight: bold; text-align: center; background-color: #f5f5f5; }
    .section-title { text-align: center; font-weight: bold; background-color: #f5f5f5; }
    .input-cell { min-height: 35px; }
    .activity-cell { width: 150px; }
    .teaching-row td { vertical-align: top; padding: 6px; }
    b, strong { font-weight: bold; }
</style>
```

### CSS类用途
- `label-cell`：左侧标签列，宽度90px，加粗居中，灰色背景
- `section-title`：区块标题行（colspan=6），居中加粗，灰色背景
- `input-cell`：内容区域，最小高度35px
- `activity-cell`：教师活动/学生活动/设计意图列，宽度150px
- `teaching-row`：教学实施环节行，顶部内边距6px

## 七、生成流程

1. **读取配置**：询问用户教案配置（课程名称、课题名称、教学目标维度、可选模块等）
2. **参考模板**：打开 `通用学科-教学设计参考模板.html` 查看结构
3. **填充内容**：按照上述表格顺序逐行填充
4. **可选模块处理**：根据用户配置决定是否添加模块A、模块B的行
5. **生成文件**：保存为HTML文件到 `.\`
6. **告知导出Word方法**

---

## 八、Word导出方法

将HTML文件复制为`.doc`扩展名，Word会自动识别并保持格式：

```cmd
copy ".\{文件名}.html" ".\{文件名}.doc"
```

然后用Word打开，文件 → 另存为 → 选择 `.docx` 格式。

## 十、注意事项

1. 所有物理量/数学符号使用下标，如 `X₁`、`a_n`、`I_m`
2. 公式中的乘号使用 `·` 或省略，不用 `*`
3. 时间分配要合理，总计约40-45分钟/课时
4. 教学目标维度名称和数量根据用户配置灵活调整
5. 可选模块根据用户配置决定是否包含
6. 教学内容要与对应学科的教材匹配
7. 德育/思政元素要自然融入，不要强行说教
8. `<br>` 用于换行，保持HTML紧凑
9. 每个section-title行单独使用一行tr，colspan=6，居中加粗灰色背景
10. `activity-cell` 宽度为150px，对应教师活动、学生活动、设计意图三列
